Problem framing

Ability to identify the right problem to solve, not just solve problems well

Project: SimpleNexus Agent Experience

Problem

SimpleNexus dominated digital mortgage solutions for lenders but was missing real estate agents - a key influencer in which lenders their buyers choose. The challenge was expanding into a new user base without disrupting the core business or building features based on assumptions.

Approach

I immersed myself in the real estate world through licensing courses, agent interviews, and open houses. Rather than asking “what features do agents want,” I identified the underlying problem: a communication breakdown around pre-qualification letters that hurt agents, lenders, and borrowers.

Solution

I led a 5-day design sprint that created an agent dashboard where agents could generate pre-qual letters with borrower-friendly language, parameters stayed within lender guidelines while giving agents control, and borrowers could self-serve updates within agent-defined limits.

Impact

Successfully brought a new user segment into the SimpleNexus ecosystem while strengthening relationships between existing lender clients and their agent partners. The solution identified the right problem before committing significant development resources.

Our team collaborated over 5 intense days and learned a lot about our users through interviews and sharing notes together.

The agent dashboard solved the real problem - communication friction around pre-qualification - rather than just adding 'agent features' to the existing product.
